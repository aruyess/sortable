🦹 Sortable — как это работает

Разбор проекта «с головы»: что происходит, когда браузер открывает index.html, и как устроен поток данных.

0. Точка входа — браузер читает HTML

Браузер загружает index.html, строит DOM (пустую таблицу, toolbar, модалку), затем натыкается на строку:

<script src="app.js"></script>

Весь JS-файл исполняется сверху вниз: объявляются константы (API, COLUMNS, SEARCH_FIELDS, STRING_OPS, NUMBER_OPS), создаётся объект state со значениями по умолчанию, определяются все функции. В самом конце файла — вызов init(). Отсюда начинается жизнь приложения.

1. Функция запуска — init()

const init = async () => {
  try {
    loadURL();                                // 1. читаем URL в state
    const response = await fetch(API);        // 2. тянем JSON
    const heroes = await response.json();
    state.all = heroes;                       // 3. сохраняем в state
    syncControlsFromState();                  // 4. выставляем UI
    bindEvents();                             // 5. вешаем обработчики
    applyFilter();                            // 6. фильтр → сорт → рендер

    if (state.heroSlug) {                     // 7. если в URL был hero — открыть
      const hero = state.all.find((h) => h.slug === state.heroSlug);
      if (hero) openDetail(hero);
    }
  } catch (err) { /* показываем ошибку в таблице */ }
};

init();

Разберём каждый из 7 шагов.

Шаг 1 — loadURL(). Если пользователь пришёл по ссылке ?q=cat&sort=name&dir=desc, мы должны восстановить это состояние до загрузки данных. Функция parseQuery через split("&") + reduce превращает query-строку в объект { q: "cat", sort: "name", dir: "desc" }, а loadURL раскладывает его в state.
Шаг 2 — await fetch(API). Запрашиваем all.json с rawcdn.githack.com. fetch возвращает Promise. await приостанавливает функцию (async-функция), пока ответ не пришёл. Затем response.json() парсит JSON — тоже через await.
Шаг 3 — state.all = heroes. Массив из ~731 героя лежит в памяти. Больше ни одного запроса делать не надо — все операции идут в браузере.
Шаг 4 — syncControlsFromState(). Выставляет value у input поиска, у селекта размера страницы, и заполняет селекты полей/операторов через populateFieldSelect и populateOpSelect.
Шаг 5 — bindEvents(). Один раз навешивает слушатели на search, searchField, searchOp, pageSize, prev, next, closeDetail и backdrop модалки. Каждое событие обновляет state и вызывает applyFilter() или render().
Шаг 6 — applyFilter(). Это ядро приложения (см. раздел ниже).
Шаг 7 — автооткрытие детали. Если в URL был ?hero=batman, ищем героя по slug и открываем модалку.

2. Главный конвейер — event → state → filter → sort → render → URL

Любое изменение (ввод в поиск, клик по заголовку, смена страницы) идёт по одному пути:

событие │ ▼ обновляется state │ ▼ applyFilter() ──► state.filtered = state.all.filter(makeMatcher(...)) │ ▼ applySort() ──► state.filtered.sort(makeCompare(...)) │ ▼ render() ──► renderHead() + renderBody() + renderPager() │ ▼ syncURL() ──► history.replaceState(...)

Такая архитектура называется unidirectional data flow — данные текут в одну сторону, и UI всегда отражает state.

3. Фильтрация — makeMatcher

Это функция-фабрика (замыкание). Она принимает выбранное поле, оператор и строку запроса, а возвращает предикат (hero) => bool, который потом передаётся в .filter.

const makeMatcher = (field, op, rawQuery) => {
  const q = rawQuery.trim();
  if (!q) return () => true;                 // пустой запрос = всё пропускаем

  if (field.type === "string") {
    const needle = q.toLowerCase();
    return (hero) => {
      const val = String(getPath(hero, field.key)).toLowerCase();
      if (op === "include") return val.includes(needle);
      if (op === "exclude") return !val.includes(needle);
      if (op === "equal")   return val === needle;
      if (op === "fuzzy")   return fuzzyMatch(val, needle);
    };
  }
  // для чисел: парсим "78 kg" в 78 и сравниваем =, ≠, >, <
};

Fuzzy-поиск — символы needle встречаются в haystack в том же порядке, но необязательно подряд. «btm» находит «Batman».

4. Сортировка — makeCompare

Тоже фабрика-замыкание. Возвращает компаратор для Array.prototype.sort.

const makeCompare = (key, dir, type) => {
  const mult = dir === "asc" ? 1 : -1;
  return (a, b) => {
    let va = getPath(a, key);
    let vb = getPath(b, key);

    if (type === "measure") {                  // "78 kg" → 78
      va = parseNumber(normalizeMeasure(va));
      vb = parseNumber(normalizeMeasure(vb));
    }

    // пропуски всегда в конец, независимо от направления
    if (va === null) return 1;
    if (vb === null) return -1;

    if (va < vb) return -1 * mult;
    if (va > vb) return  1 * mult;
    return 0;
  };
};

Ключевая деталь: missing-значения всегда идут в конец. Для этого мы возвращаем 1 / -1 для null БЕЗ умножения на mult.

5. Рендер — три функции

ФункцияЧто делает
renderHead() Рисует <th> для каждой колонки. Если колонка sortable — вешает click → onSort и, если это текущая сортировка, показывает стрелку ▲/▼.
renderBody() Берёт только текущую страницу через slice(start, start+size). Для каждого героя создаёт <tr> с кликом по карточке, внутри — <td> с картинкой / числом / строкой.
renderPager() Обновляет текст «Page N of M», включает/выключает кнопки Prev/Next, пишет счётчик найденных героев.

Почему это быстро: мы не рисуем все 731 строку в DOM. На странице максимум 100 строк (или All). При поиске мы фильтруем массив в памяти и перерисовываем только tbody.

6. URL-синхронизация

Каждый render() в конце зовёт syncURL(). Та собирает значения из state, которые отличаются от дефолтов, и пишет в URL через history.replaceState:

/?q=300&field=appearance.weight&op=gt&sort=powerstats.strength&dir=desc

На входе parseQuery разбирает query-строку обратно через .split("&").reduce(...) — обходимся без URLSearchParams. Это нужно для Piscine-совместимости: используем только пройденные приёмы (split, reduce, String).

7. Детальный просмотр — openDetail(hero)

При клике по строке таблицы:

  1. сохраняем hero.slug в state.heroSlug → попадает в URL;
  2. в #detailBody рисуем большую картинку hero.images.lg, имя, полное имя и список всех полей;
  3. добавляем класс open на #detail — CSS показывает модалку с backdrop.

Закрытие — по крестику или клику на backdrop. При этом heroSlug стирается, URL обновляется.

Коротко — одним абзацем

Браузер читает index.html, подключает app.js, внизу файла стартует init(). Она: читает URL → fetch JSON → настраивает UI → вешает события → запускает конвейер filter → sort → render → URL. Дальше любое действие пользователя обновляет state и прогоняет тот же конвейер заново. Никаких фреймворков, никакого лишнего перерендера — только текущая страница.
