# 🦹 Sortable · Superhero Database

Веб-приложение для «злодея», которому нужно удобно просматривать и фильтровать досье на супергероев. Данные загружаются из публичного [Superhero API](https://rawcdn.githack.com/akabab/superhero-api/0.2.0/api/all.json).

Написано на **чистом JavaScript** — без React, Vue, Svelte, jQuery и любых других фреймворков/библиотек.

---

## 📸 Возможности

### Обязательное
- Таблица с нужными колонками: иконка, имя, полное имя, powerstats (INT / STR / SPD / DUR / POW / CMB), раса, пол, рост, вес, место рождения, принадлежность.
- **Пагинация** с выбором размера страницы: 10 / 20 / 50 / 100 / All (по умолчанию — 20).
- **Живой поиск** — результаты обновляются после каждого нажатия клавиши.
- **Сортировка по любой колонке**:
  - первый клик — по возрастанию, повторные — переключение asc/desc;
  - числовые колонки (вес, рост, powerstats) сортируются как числа: `75 kg` идёт раньше `100 kg`;
  - пропущенные значения всегда в конце, независимо от направления.

### Бонусы
- **Поиск по любому полю**: Name / Full Name / Race / Gender / Place of Birth / Alignment / Publisher / Height / Weight / Intelligence / Strength / Speed / Durability / Power / Combat.
- **Кастомные операторы**:
  - для строк: `include`, `exclude`, `fuzzy`, `equal`;
  - для чисел: `=`, `≠`, `>`, `<`.
- **Детальный просмотр** — клик по строке открывает модалку с большим изображением и всеми полями героя.
- **URL-синхронизация** — поиск, сортировка, страница и открытый герой отражаются в query-строке. Скопированная ссылка открывается в том же состоянии.
- **Быстрый рендер** — данные грузятся один раз, на страницу отрисовывается только срез (`Array.prototype.slice`).

## 🚀 Запуск

Поскольку используется `fetch` на внешний ресурс, браузеры блокируют запросы с `file://`. Запуск через локальный HTTP-сервер:

```bash
# Python 3
python3 -m http.server 8080

# или Node.js
npx serve .
```

Затем открыть [http://localhost:8080](http://localhost:8080).

## 🧠 Архитектура

Приложение построено вокруг одного объекта `state`, который описывает текущий UI:

```js
const state = {
  all,        // все герои (загружаются один раз)
  filtered,   // отфильтрованный + отсортированный срез
  sortKey, sortDir,
  page, pageSize,
  query, field, op,
  heroSlug,   // открытый в модалке герой
};
```

Любое изменение состояния идёт по конвейеру:

```
событие → обновление state → applyFilter() → applySort() → render() → syncURL()
```

Ключевые функции:

| Функция | Что делает |
|---|---|
| `getPath(obj, "biography.fullName")` | безопасный доступ по вложенному пути (через `reduce`) |
| `parseNumber("78 kg")` | вынимает число из строки через regex |
| `makeMatcher(field, op, q)` | замыкание, возвращающее предикат для `.filter` |
| `makeCompare(key, dir, type)` | замыкание, возвращающее компаратор для `.sort` |
| `buildQuery / parseQuery` | ручная сериализация URL query без `URLSearchParams` |

## 🔗 URL-параметры

| Параметр | Значение |
|---|---|
| `q`      | строка поиска |
| `field`  | ключ поля поиска (`name`, `appearance.weight`, …) |
| `op`     | оператор (`include`, `exclude`, `fuzzy`, `equal`, `eq`, `neq`, `gt`, `lt`) |
| `sort`   | ключ колонки сортировки |
| `dir`    | `asc` / `desc` |
| `size`   | размер страницы (`10`, `20`, `50`, `100`, `all`) |
| `page`   | номер страницы |
| `hero`   | slug открытого героя |

Пример: `?q=300&field=appearance.weight&op=gt&sort=powerstats.strength&dir=desc`

## 🧪 Как проверить

1. Открыть страницу — таблица отсортирована по `Name` по возрастанию, 20 строк.
2. Набрать `Cat` в поле поиска — появятся Catwoman, Black Cat и т.д.
3. Сменить поле на **Weight**, оператор на **>**, ввести `300` — останутся только тяжеловесы.
4. Дважды кликнуть по заголовку **Place of Birth** — сортировка по убыванию.
5. Кликнуть по любому герою — откроется модалка с деталями.
6. Скопировать URL в новую вкладку — состояние восстановится.
